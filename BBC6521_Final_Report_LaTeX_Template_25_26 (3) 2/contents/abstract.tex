Modern AI agents rely heavily on external tools, but on Linux today the operating system plays no part in how those calls are gated, and the entire control plane sits in a user-space gateway.
That leaves a set of properties one would normally expect the kernel to guarantee resting instead on a single daemon: that the executed tool actually matches the one that was approved, that requests stay bound to the session that opened them, and that approval state survives a crash.

We present \texttt{linux-mcp}, a kernel-assisted control plane for LLM tool invocation on Linux.
Semantic parsing and tool execution stay in user space; the kernel module holds only a narrow slice of control-plane state (which tools and agents are registered, which approval tickets are live, and which session a request belongs to).
This design requires no modification to existing tools beyond a declarative JSON manifest and a corresponding adapter layer, and makes approval state durable across daemon failure and protected from unprivileged user-space interference.

We evaluate \texttt{linux-mcp} against a pure user-space baseline and a hardened seccomp variant, covering performance, behaviour under attack, and behaviour after daemon failure.
The kernel module blocks every attack in our targeted control-plane test suite, against an 84.6\% bypass rate for the user-space baseline, while keeping end-to-end latency within measurement noise at typical payloads.
An extended-attack rerun further surfaced a missing peer-credential check in the stock module, closed without measurable cost.
The prototype is publicly available at \url{https://github.com/xiheng724/linux_mcp}.
