Modern AI agents depend heavily on external tools, yet on Linux today the operating system plays no part in gating those tool calls: the entire control plane lives in a user-space gateway.
As a result, several properties that would ordinarily fall within the kernel's responsibility are left to a single daemon: ensuring that the executed tool corresponds to the approved tool, preserving the binding between requests and the sessions that opened them, and maintaining approval state across crashes.

To address these challenges, we present linux-mcp, a kernel-assisted control plane for LLM tool invocation on Linux.
Semantic parsing and tool execution stay in user space; the kernel module holds only a narrow slice of control-plane state (which tools and agents are registered, which approval tickets are live, and which session a request belongs to).
This design requires no modification to existing tools beyond a declarative JSON manifest and a corresponding adapter layer, and makes approval state durable across daemon failure and protected from unprivileged user-space interference.

We evaluate linux-mcp against a pure user-space baseline and a hardened seccomp variant along three axes: performance, behaviour under attack, and behaviour after daemon failure.
The kernel module blocks every attempt in our targeted control-plane attack suite, against an 84.6\% bypass rate for the user-space baseline, while end-to-end latency at typical payloads remains within measurement noise.
An extended-attack rerun additionally identified a missing peer-credential check in the stock module, which was closed at no measurable cost.
The prototype is publicly available at \url{https://github.com/xiheng724/linux_mcp}.
