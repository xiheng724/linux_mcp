AI agents 越来越多地依靠调用外部工具来完成任务，但在今天的 Linux 上，操作系统本身并不参与这些调用的仲裁，整条控制平面都跑在用户态的 gateway 进程里。
这样一来，若干本应由内核保证的性质，最终都只能依赖于这一个守护进程的正确和存活：被执行的工具是否就是被批准的那一个、请求是否仍然绑定在它所属的会话上、批准状态能否在进程崩溃之后继续有效。

为此，我们提出了 \texttt{linux-mcp}，一种面向 Linux 的、内核辅助的工具调用控制平面。它并不把完整的工具逻辑搬进内核，而是把 JSON 解析和工具执行继续留在用户态，只将一小块控制平面状态下沉进一个内核模块：哪些工具和 agent 已经注册、哪些批准票据仍然有效、某个请求归属于哪一个会话。
现有工具无需任何代码改动，只需补一份声明式的 JSON manifest；批准状态因此可以跨越守护进程的崩溃继续保留，也不会被非特权用户态进程干扰。

我们将 \texttt{linux-mcp} 与纯用户态基线以及 seccomp 强化版基线做了对比，评测同时覆盖性能层面和抗攻击、故障恢复两类行为。
实验中，内核模块阻断了测试集中的全部控制平面攻击，而两种用户态基线分别存在 90.9\% 和 36.4\% 的绕过率；扩展攻击重跑还暴露出原始模块中一处缺失的 peer-credential 校验，以六行补丁修复，没有可测的延迟代价。性能开销也不大：小、中负载下延迟与用户态无显著差异，1\,MB 下反而约低 3.4\%；配对微基准显示，完整的内核仲裁路径仅比 Generic Netlink 底噪多 \num{4.334}~$\mu$s。
该原型在 14 个应用中暴露 43 个工具，已开源：\url{https://github.com/xiheng724/linux_mcp}。
